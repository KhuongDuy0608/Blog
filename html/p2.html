CÁC BIẾN VAR, LET, CONST TRONG JAVASCRIPT

Trong JavaScript, việc quản lý và sử dụng biến là một yếu tố quan trọng để xây dựng các ứng dụng hiệu quả và dễ bảo trì. Trước đây, var là cách duy nhất để khai báo biến, nhưng từ khi ES6 ra đời, hai từ khóa mới là let và const đã được bổ sung để giúp lập trình viên dễ dàng kiểm soát phạm vi (scope) và tính chất không thay đổi của các biến trong chương trình. Mỗi loại biến này có đặc điểm và ứng dụng riêng, điều này ảnh hưởng trực tiếp đến cách thức chúng ta viết mã và xử lý các vấn đề liên quan đến phạm vi và khả năng thay đổi giá trị. Bài viết này sẽ phân tích sự khác biệt giữa var, let, và const, và giải thích khi nào nên sử dụng mỗi loại trong các tình huống khác nhau.

1. Khai báo biến với var, let, và const

JavaScript cung cấp ba cách khai báo biến: var, let, và const. Mỗi cách có đặc điểm và phạm vi hoạt động khác nhau.

// Sử dụng var
var x = 10;
console.log(x); // 10

// Sử dụng let
let y = 20;
console.log(y); // 20

// Sử dụng const
const z = 30;
console.log(z); // 30

2. Tính chất Hoisting

Biến được khai báo bằng var sẽ được "hoisted" (nâng lên đầu phạm vi) nhưng không khởi tạo giá trị. Với let và const, biến cũng được hoisted nhưng không thể sử dụng trước khi khai báo.

console.log(a); // undefined
var a = 10;

// console.log(b); // Lỗi: Cannot access 'b' before initialization
let b = 20;

3. Phạm vi biến (Scope)

Phạm vi của biến phụ thuộc vào cách khai báo:

var: phạm vi toàn cục hoặc hàm.

let và const: phạm vi khối (block scope).

if (true) {
    var globalVar = "Toàn cục";
    let blockVar = "Khối";
    const blockConst = "Khối";
}

console.log(globalVar); // "Toàn cục"
// console.log(blockVar); // Lỗi: blockVar không được định nghĩa
// console.log(blockConst); // Lỗi: blockConst không được định nghĩa

4. Biến trong Template Literals

JavaScript hỗ trợ sử dụng biến trong chuỗi thông qua Template Literals, giúp code ngắn gọn và dễ đọc hơn.

const name = "JavaScript";
const version = 2023;
console.log(`Ngôn ngữ: ${name}, Phiên bản: ${version}`);
// Output: Ngôn ngữ: JavaScript, Phiên bản: 2023

5. Biến động (Dynamic Typing)

JavaScript là ngôn ngữ "dynamic typing", nghĩa là một biến có thể thay đổi kiểu dữ liệu trong suốt quá trình chạy.

let dynamicVar = 100; // kiểu số
console.log(typeof dynamicVar); // "number"

dynamicVar = "Hello"; // kiểu chuỗi
console.log(typeof dynamicVar); // "string"

Kết luận

Biến trong JavaScript rất linh hoạt và mạnh mẽ, nhưng cũng cần sử dụng đúng cách để tránh lỗi. Hãy làm quen với let và const, bởi chúng an toàn hơn so với var trong hầu hết các trường hợp. Chúc bạn học JavaScript vui vẻ và hiệu quả!
